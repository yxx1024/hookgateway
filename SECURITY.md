# 安全政策

## 支持范围
- 维护中的主分支（master）与最新发布版本。

## 漏洞报告
- **请勿**在公开 Issue 中披露安全漏洞细节。
- 推荐使用 GitHub Security Advisories 进行私密报告。
- 若仓库未启用安全通道，请通过代码托管平台私信维护者并提供复现步骤。

## 响应与修复
- 72 小时内确认收到报告。  
- 7 天内给出初步评估与缓解建议。  
- 修复周期（视复杂度与影响范围而定）：  
  - 严重/高危：14–30 天  
  - 中危：60–90 天  
  - 低危：尽力而为

## 协调披露
修复完成后与报告者协商披露时间与公告内容。
